# 公開確認チェックリスト

## 公開前
- `index.html` を GitHub Pages の公開対象にしていることを確認する
- `.env`、秘密鍵、認証情報、外部サービスの秘密トークンが含まれていないことを確認する
- `node_modules/` や `dist/` など不要ファイルが公開対象に入っていないことを確認する

## GitHub Pages 設定
- 対象リポジトリの `Settings > Pages` を開く
- 公開元のブランチとフォルダが、実際に `index.html` を置いた場所になっていることを確認する
- 反映直後は数分待ってから公開URLを開く

## ブラウザ確認
- 通常ウィンドウで公開URLを開く
- シークレットウィンドウでも公開URLを開く
- ハードリロードを実行する
  - Windows: `Ctrl + F5`
- 旧キャッシュが疑わしい場合は、ブラウザキャッシュを削除して再確認する

## 画面確認
- 初回表示でダッシュボードが開く
- 歩留グラフが表示される
- 可動率グラフが表示される
- 右上の `データ入力画面へ` からモード選択画面へ移動できる
- `歩留入力` を開ける
- `可動率入力` を開ける
- `トップへ戻る` でダッシュボードに戻れる

## ダミーデータ確認
- 今月の履歴に歩留データが10件前後入っている
- 今月の履歴に可動率データが10件前後入っている
- 履歴画面でデータを開いてもエラーにならない
- ダッシュボードの円グラフ・推移グラフが空になっていない

## 反映されない場合の確認順
1. GitHub に最新の `index.html` が push されているか確認する
2. GitHub Pages の公開元が正しいか確認する
3. 公開URLをシークレットウィンドウで開く
4. `Ctrl + F5` で再読み込みする
5. それでも古い画面なら、Pages の再デプロイ完了を待つ
